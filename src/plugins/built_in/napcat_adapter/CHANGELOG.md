# Changelog

## [1.3.2] - 2026-07-10

### 开发侧

- `_manifest.json` 声明 `plugin_type: "adapter"`，使 Host（1.0.12 起）的适配器独立 Runner 分组机制对本适配器生效：适配器不再与第三方插件同组，避免第三方插件 Runner 重启时 QQ 连接被一并中断。该字段自 Host 1.0.0 起即可识别，不影响旧版本加载。

## [1.3.0] - 2026-07-01

### 用户感知功能

- 新增 NapCat 通知事件逐类型开关，可分别控制戳一戳、撤回、禁言、表情回应、群文件、入群退群、管理员变动、精华消息和群名变更等通知是否注入 Host。
- 默认过滤未显式启用的通知类型，避免 `input_status` 等高频噪音通知刷屏。

## [1.2.0] - 2026-06-26

### 用户感知功能

- 补充 notice 事件的聊天黑白名单过滤，避免黑白名单外的群聊或私聊触发戳一戳等通知响应。
- 保留小程序卡片消息的元数据，减少转发或处理卡片消息时的信息丢失。

### 开发侧

- 更新 `_manifest.json`，同步插件版本与兼容性声明。
- 调整列表模式校验逻辑，按模式区分处理消息过滤。
